# 習題 3 參考解 — 糾正「p=0.20 代表 80% 機率假設為真」

> **錯誤說法**:「檢定算出 p=0.20,所以這個假設有 80% 的機率是真的。」

這句話把兩個不同的東西偷換了。糾正如下:

**1. p-value 的定義方向是反的。**
p-value 是 `P(看到這麼極端或更極端的資料 | H₀ 為真)`,也就是
**「假設成立的前提下,資料的機率」**。它**不是** `P(H₀ 為真 | 資料)`——
不是「資料給定下,假設的機率」。把前者當後者,等於把條件機率的方向倒過來。

**2. 1 − p 也不是「假設為真的機率」。**
「p=0.20 ⟹ 80% 為真」這步暗示 `P(H₀|資料) = 1 − p`,這完全沒有根據。
要從 `P(資料|H₀)` 得到 `P(H₀|資料)`,必須用 Bayes:

```
P(H₀|資料) = P(資料|H₀) · P(H₀) / P(資料)
```

少了 **先驗 `P(H₀)`(base rate)** 和分母 `P(資料)`,根本算不出「假設為真的機率」。
忽略 base rate 正是典型的 base-rate fallacy:在一個「絕大多數型態都是雜訊」的領域裡
(先驗就很低),即使 p 不大,該型態為真的後驗機率仍可能很低。

**3. p=0.20 在實務上的意思。**
它代表「就算 H₀(沒效果)為真,純靠運氣也有 20% 機會看到這麼極端的結果」。
這通常**不足以**拒絕 H₀。再加上小樣本(n=9)power 很低,p 大也可能只是 power 不夠,
而非真的沒效果——**「沒拒絕」既不能證明 H₀,也不能反過來給 H₀ 一個高機率**。

**一句話**:p 衡量的是「資料在 H₀ 下有多意外」,不是「H₀ 有多可能」;要講後者,
得帶著先驗做 Bayesian 推論(Stage 9 的 posterior / credible interval)。

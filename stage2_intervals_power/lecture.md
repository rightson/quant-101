# Stage 2 精講稿 — Interval Estimation 與 Hypothesis Testing(含 Power)★

主旨三句:
1. **信賴區間**是「程序」的性質,不是某一條區間的機率;比例的小樣本要用 Wilson,不是 Wald。
2. **p-value** 不是「假設為真的機率」;它是「假設為真時,看到這麼極端資料的機率」。
3. **Power** 是「真有效果時你抓得到的機率」;n=9 的 power 低到「沒拒絕」毫無資訊量。

---

## 1. Confidence interval 的正確詮釋(M2.1)

一個 95% CI 的正確意思:**如果你反覆抽樣、每次用同一套程序造區間,長期會有約 95%
的區間蓋住真值。** 覆蓋率是「程序」的性質。

❌ 錯誤詮釋:「真值有 95% 機率落在這條(已經算出來的)區間裡。」
真值是固定的(雖然未知);算出來的區間也是固定的;「機率」描述的是**還沒抽樣前的程序**,
不是事後這一條。要講「真值的機率分布」,你得用 Bayesian 的 credible interval(Stage 9)。

對判讀的用途:當文章給你一個點估計,你心裡要把它換成「一條會抖的區間」,並記得
**寬度才是重點**。n=9 的區間寬到沒有結論能站住。

## 2. 比例區間:為何 Wald 會壞、Wilson 對(M2.2)

文章說「創新高後 9 次有 8 次怎樣怎樣 = 88%」。要對 `p̂ = 8/9` 配區間。

### 2.1 Wald(教科書最常見、也最常錯)

```
p̂ ± z·sqrt(p̂(1-p̂)/n)
```

對 8/9:`0.889 ± 1.96·sqrt(0.889·0.111/9)` = `0.889 ± 0.205` = **[0.684, 1.094]**。
**上界 1.094 > 1**,是個不可能的機率。Wald 在
(a) n 小、(b) p̂ 靠近 0 或 1 時覆蓋率嚴重不足、區間還會溢出 [0,1]。8/9 同時踩中兩個雷,
**所以這裡用 Wald 是錯的**。

### 2.2 Wilson(正解)

把 score 檢定 `|p̂-p| / sqrt(p(1-p)/n) = z` 對 p 解二次方程,得:

```
center = (p̂ + z²/2n) / (1 + z²/n)
half   = (z / (1+z²/n)) · sqrt( p̂(1-p̂)/n + z²/4n² )
```

對 8/9:**Wilson CI = [0.565, 0.980]**。永遠落在 [0,1] 內,小樣本覆蓋率接近名目值。
直覺:Wilson 把中心往 0.5 收縮(像加了一點先驗的虛擬觀測),修正了 Wald 在邊緣的過度自信。

### 2.3 其他兩個與時機

- **Agresti–Coull**:「加 z²/2 個成功與失敗再做 Wald」,簡單、行為接近 Wilson。
- **Clopper–Pearson(exact)**:用 Beta 分布,保證覆蓋率 ≥ 名目(偏保守、較寬)。當你
  要「寧寬勿假」的保證時用。

**判讀重點**:8/9 的 88% 聽起來很猛,但 Wilson CI 從 56.5% 一路到 98%——
「真實命中率可能只有五成多」並沒有被資料排除。一個寬到這種程度的區間,等於沒有結論。

## 3. Hypothesis testing 與 p-value 的真正定義(M2.3)

- **H₀(虛無)/ H₁(對立)**:H₀ 通常是「沒有效果」。
- **Type I error(α)**:H₀ 為真卻拒絕(假陽性)。一般訂 α=0.05。
- **Type II error(β)**:H₁ 為真卻沒拒絕(假陰性)。**Power = 1 − β**。
- **p-value 的定義**:*假設 H₀ 為真*,得到「跟觀測一樣極端或更極端」資料的機率。

常見三大誤解(務必能當場糾正):
1. ❌「p 是 H₀ 為真的機率」。→ p 是條件在 H₀ 下的資料機率,不是 H₀ 的機率。
2. ❌「p=0.20 代表 80% 機率假設為真」。→ 見 `solutions/ex3_pvalue_misreading.md`,
   把 P(資料|H₀) 倒過來當成 P(H₀|資料) 是 base-rate fallacy。
3. ❌「沒拒絕 = 證明 H₀」。→ **沒拒絕可能只是 power 太低**;這正是 n=9 的處境。

## 4. 核心:Power 與 sample size(M2.4)

### 4.1 power 是什麼

**Power = 當效果真的存在時,你的檢定成功拒絕 H₀ 的機率。** 它取決於
(a) 效果大小(effect size)、(b) 樣本數 n、(c) α、(d) 變異。effect size 用 Cohen's d:

```
d = (μ₁ - μ₀) / σ
```

### 4.2 文章的 case:n=9 偵測 10pp mean 差異

文章拿「事件後 1Y 中位 −1.7%」對「buy-and-hold +8.5%」比,差約 **10.2pp**。
年報酬 σ 取 TAIEX 量級 ~22%,於是:

```
d = 0.102 / 0.22 ≈ 0.464
```

用 one-sample t-test(非中心 t,ncp = d·√n)算 power:

| n | power(d=0.464, α=0.05 雙尾) |
|---|---|
| **9** | **≈ 0.233** |
| 39 | ≈ 0.80 |

**n=9 的 power 只有 ~0.23**:就算「先出場真的比較好」這個效果存在,你用 9 筆資料也
只有不到四分之一的機率能偵測到它。所以「我回測沒看到顯著差異」在 n=9 下**完全不是
證據**——你本來就抓不到。要把 power 拉到 0.8,需要 **n ≈ 39**(約 39 次獨立事件,
而文章只有 9 次,還彼此相關,見 Stage 5)。

### 4.3 把它畫出來:effect size × n → power 熱力圖

notebook 會掃 `d ∈ [0.1, 1.2] × n ∈ [5, 80]` 算每格 power,標出 n=9 的那一行——
你會看到 n=9 那整行幾乎全是低 power 的冷色,只有在 d 大得不切實際時才勉強變熱。
一張圖把「小樣本沒有 power」變成肌肉記憶。

### 4.4 power 與 CI 是一體兩面

低 power 等價於「寬到沒用的 CI」:兩者都在說「這份資料分辨不出有沒有效果」。
所以 gate 同時要你做 Wilson CI(寬到 [0.565, 0.980])**和** power(0.23)——它們是同一件事的
兩種講法。

## 5. 一頁回顧

| 概念 | 重點 | 文章 case |
|---|---|---|
| CI 詮釋 | 95% 是程序長期覆蓋率,不是這條區間的機率 | 點估計要換成會抖的區間 |
| Wald 壞掉 | 小 n / 極端 p̂ ⟹ 溢出 [0,1]、覆蓋不足 | 8/9 → [0.684, 1.094] ❌ |

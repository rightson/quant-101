# Stage 3 精講稿 — Multiple Testing 與 Research Methodology ★

主旨三句:
1. 你做的不是一個檢定,是一**家族**檢定;α=0.05 是「每個」的錯誤率,不是「整批」的。
2. 隱性比較(implicit comparisons)來自 researcher degrees of freedom——那些沒被講出來的
   參數選擇;每多一個,「找到顯著」的機率就膨脹。
3. 校正方法分兩派:控 **FWER**(一個假陽性都不要,Bonferroni/Holm)與控 **FDR**
   (容忍固定比例假陽性,Benjamini-Hochberg)。校正後,文章那種型態通常活不下來。

---

## 1. 為什麼「多重」會出事(M3.4 的直覺先講)

單一檢定在 H₀ 為真時,有 α=5% 機率假陽性。做 `m` 個**獨立**檢定、且全部 H₀ 為真:

```
P(至少一個假陽性) = 1 − (1−α)^m
```

| m(比較數) | P(≥1 假陽性) |
|---|---|
| 1 | 0.05 |
| 14 | 0.51 |
| 20 | 0.64 |
| 100 | 0.99 |
| 1024 | ≈ 1.000 |

**只要試夠多組,撈到一個 p<0.05 是必然的。** 這跟有沒有真訊號完全無關。所以「我回測
發現一個 88% 命中的型態」這句話,在不知道他試了幾組之前,資訊量接近零。

## 2. Researcher degrees of freedom 與隱性比較(M3.2)

**Researcher degrees of freedom**:分析過程中每一個「可以這樣、也可以那樣」的選擇。
文章的事件定義就藏了一整排:

| 自由度 | 文章用的值 | 但其實可以是… |
|---|---|---|
| 新高 lookback | 一年 | 半年?9 個月?1.5 年? |
| 急殺判定窗口 | 3–4 天 | 2?3?4?5 天? |
| 近期跌幅窗口 | 近三日 | 1?2?3?5 日? |
| 跌幅門檻 | 前 2% | 1%?2%?5%?10%? |
| 評估報酬 horizon | (隱含) | 1 月?3 月?6 月?1 年? |

就算每個只列 4 個合理選項,組合數 = `4⁵ = 1024`。**作者(可能不自覺地)在 1024 條
分析路徑裡,挑了出數字最漂亮的那一條來講。** 這就是 specification mining。

### 2.1 p-hacking vs HARKing vs garden of forking paths

- **p-hacking**:刻意試到 p<0.05 才停(加減樣本、換定義、挑子區間)。
- **HARKing**(Hypothesizing After the Results are Known):先看到結果,再把假設「框」成
  剛好解釋它的樣子,然後當成事前假設報告。文章「先看到創新高後有幾次急殺,再把
  『創新高+急殺』定義成型態」就是典型 HARKing。
- **Garden of forking paths**(Gelman & Loken 2013):**最隱蔽的一種**。研究者甚至沒有
  真的跑 1024 組——他只跑了一組。但**如果資料長得不一樣,他會做不同的選擇**。所以那條
  路徑是「資料相依」的,等效於在所有可能路徑裡做了選擇。**不需要實際 p-hack,光是
  『會看資料決定定義』就足以讓單一 p 值失去意義。**

### 2.2 為什麼 pre-registration 有用(M3.3)

Pre-registration:**在看資料前**,把假設、定義、檢定方法全部寫死、公開。
科學哲學上,它把「探索」(generate hypotheses)與「驗證」(test them)分開:

- in-sample、看過資料後框出來的顯著性 → 只是描述這份資料,不能外推(它已經用掉了
  資料來「挑路徑」)。
- pre-registered 的顯著性 → 才是對未來的真預測,因為路徑在看資料前就固定了。

文章沒有 pre-register,且定義明顯是回頭框的,所以它的「顯著」是 in-sample 的、貶值的。
真正的檢驗在 **out-of-sample / 真實前瞻**(Stage 9 的 backtest overfitting 會接這條)。

## 3. 校正方法:FWER 與 FDR(M3.1)

設 `m` 個檢定、p 值 `p₁,…,p_m`。

### 3.1 控 FWER(family-wise error rate = P(≥1 假陽性))

- **Bonferroni**:拒絕 `p_i ≤ α/m`。簡單、保守。1024 個檢定 → 門檻 `0.05/1024 ≈ 4.9e-5`。
  文章的 p 值要小到這種程度才算數。
- **Holm**(step-down):把 p 由小到大排,第 k 小的門檻是 `α/(m−k+1)`;一旦某個沒過就
  停。比 Bonferroni 有 power,且一樣嚴格控 FWER。**有 Bonferroni 就用 Holm,沒道理用前者。**

### 3.2 控 FDR(false discovery rate = E[假陽性 / 總拒絕])

- **Benjamini-Hochberg**:把 p 由小到大排,找最大的 k 使 `p_(k) ≤ (k/m)·α`,拒絕前 k 個。
  控的是「拒絕裡有多少比例是錯的」,而非「有沒有一個錯的」。**較不保守**,適合「我能
  容忍一小撮假陽性、但想多抓真陽性」的場景(例如初篩一堆因子)。

### 3.3 該用哪個

- 要「這個型態到底成不成立」的單點裁決 → FWER(Holm/Bonferroni),寧可錯殺。
- 要從一大堆候選裡篩出「值得進一步驗證」的子集 → FDR(BH)。

對文章的單一宣稱,正確做法是把它放回它所屬的 1024 個比較家族裡,用 FWER 校正——
這一放回去,它幾乎必然失去顯著性。

## 4. 接到 case:grid search 的實證(notebook 在做的事)

`notebook_stage3.py` 把事件定義參數化,對(合成、無內建訊號的)TAIEX 跑整個 grid:

1. 每組定義 → 一個事件集 → 對「事件後報酬 vs 無條件報酬」做 t 檢定 → 一個 p 值。
2. 看 raw 顯著(p<0.05)的比例:在**沒有真訊號**的資料上,它會在 ~5% 上下——這正是
   「總會撈到幾個」。
3. 看「最顯著」那組:它的數字會很漂亮,但它只是 1/N 的運氣。
4. 套 Bonferroni / Holm / BH:校正後存活的幾乎歸零。

把這張圖看進去,你以後看到任何「我試了很多參數後發現…」都會自動問:**「你試了幾組?
校正了嗎?out-of-sample 還在嗎?」**

## 5. 一頁回顧

| 概念 | 重點 | 文章 case |
|---|---|---|
| 多重比較 | `P(≥1 假陽性)=1−(1−α)^m` | 5 個 DoF → 1024 組 → P≈1 |
| Researcher DoF | 沒講出來的參數選擇 | 新高/急殺/跌幅/門檻/horizon |
| HARKing | 先看結果再框假設 | 「創新高+急殺」是回頭框的 |
| Forking paths | 不必真 p-hack,會看資料決定就夠 | 單一 p 值已失意義 |
| FWER vs FDR | Holm 控「有沒有錯」/ BH 控「錯的比例」 | 放回家族校正後不顯著 |
| Pre-registration | 看資料前固定路徑 | 文章沒做 → in-sample 貶值 |
